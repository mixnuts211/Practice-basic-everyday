## What I learned

### 📝 CSS

#### ✔️ **`checkbox`로 아코디언 토글 구현하기**

:checked 의사 클래스를 활용하면 HTML, CSS만으로 아코디언 토글 구현이 가능합니다.

1. 구현하고자 하는 아코디언 메뉴들을 `checkbox`타입을 가진 `input`으로 작성해줍니다. 여러 아코디언 메뉴들의 동시 열람을 허용할 경우 `checkbox`를, 여러 메뉴 중 하나의 항목만 열람을 허용할 경우 `radio`를 사용해주면 됩니다.
2. 각 `checkbox`를 연결할 `label`을 함께 작성해줍니다. 이 `label`은 **<u>`checkbox`의 기본 형태를 감추더라도 그 역할을 할 수 있게 도와줍니다.</u>**

```html
<!-- 작업 시 작성했던 코드 -->
<li>
  <input type="checkbox" id="toggle-q-1" class="toggle-input" />
  <label class="card-faq-text-question" for="toggle-q-1">
    How many team members can I invite?
  </label>
  <!-- checkbox 기능을 하는 input과 해당 input의 label로 for와 id명을 동일하게 해주어 label을 연결 -->
  <div class="card-faq-text-answer" id="toggle-a-1">
    You can invite up to 2 additional users on the Free plan. There is no limit
    on team members for the Premium plan.
  </div>
  <!-- div => 아코디언이 펼쳐지면 보여질 내용 -->
</li>
```

3. ~~`checkbox` 요소는 `display : none;`처리를 해줍니다.~~ 남아있는 `label`을 활용하여 원하는 화면으로 스타일링을 해줍니다.

<img width="375" alt="스크린샷 2023-07-25 오후 3 45 16" src="https://github.com/mixnuts211/Practice-basic-everyday/assets/94048689/e05b90ba-5061-4a6c-96af-74fd817a6cea">

> => **`display : none;`처리 시 checkbox의 기본 속성을 사용할 수 없습니다. 죽, 키보드로부터의 접근이 불가능해집니다. 이 경우에는 `display : none;`으로 DOM트리에서 완전히 제외하는 것이 아니라 `sr-only`(screen reader only) 기법을 사용하는 것이 더 적절합니다.**

4. `checkbox`에 부여된 클래스를 사용하여 `:checked` 의사 클래스가 적용될 시 나타내고자 하는 화면을 스타일링 해줍니다.
5. 열람하고 싶은 아코디언 메뉴의 `label`을 클릭 시, `checkbox`의 상태는 `checked`로 변경이 되며, 4번에서 부여해준 스타일을 화면에서 만날 수 있습니다.

```css
/* label이 아닌 input에 의사 클래스를 사용할 수 있습니다. */
/* input은 감추어져 있지만 label을 클릭하면 input은 `checked`의 상태를 가지게 됩니다. */

.toggle-input[type="checkbox"]:checked ~ .card-faq-text-answer {
  visibility: visible;
  height: fit-content;
  padding-bottom: 15px;
  box-sizing: border-box;
  transition-duration: 0.5s;
}

/* type이 checkbox인 .toggle-input가 checked가 된다면, .card-faq-text-answer은 상단에 작성된 스타일을 가지게 됩니다. */
```

<br>

#### ✔️ `display : none`과 `visibility : hidden`

두 속성은 '화면에서 보이지 않게 하는 것' 이라는 성질에서 비슷해보이지만 차이점이 존재합니다.

- `display : none`의 경우 렌더링 시 화면에 존재하지 않는 요소로 처리됩니다. 즉 렌더 트리 자체에서 제외됩니다. 해당 요소의 속성이 변경될 경우, 리플로우와 리페인트를 발생시킵니다.
- `visibility : hidden`의 경우 요소는 해당 위치에 존재하며 공간을 차지하지만, 화면에서 컨텐츠만을 숨깁니다. 해당 요소의 속성이 변경될 경우, 리페인트만 발생하게 됩니다.

<br>

#### ✔️ **네거티브 마진 활용하기**

네거티브 마진은 단순히 '네거티브 방향만큼 요소를 이동'한다는 개념보다 <u>'요소의 바깥으로 증감되는 마진을 네거티브하게 주게 될 경우 요소 안으로 깍이게 되는 것'</u>으로 요소가 차지할 수 있는 가용공간이 주어진 네거티브 마진 크기만큼 줄어들게 됩니다.<br>
이러한 성격을 이미지와 함께 레이아웃 배치에 활용할 수 있습니다. 다음은 작업 때 사용한 예시입니다.

1. 다음과 같이 기본적인 레이아웃을 작성해줍니다.

```html
<div class="la-wrap">
  <div class="main-banner-txt-wrap">
    <p>텍스트</p>
  </div>
  <div class="main-banner-img-wrap">
    <img src="이미지주소" alt="이미지설명" />
  </div>
</div>
```

```css
.la-wrap {
  padding: 10rem 0 15rem;
  display: flex;
  align-items: flex-start;
  gap: 2rem;
}
```

<img width="400" alt="before" src="https://github.com/mixnuts211/Practice-basic-everyday/assets/94048689/7772d29b-f261-4d2c-ac6f-0b5117e07f3b">

2. img에게 네거티브마진 `-30rem`을 부여합니다. 고정 값이 없을 경우 자식만큼 크기가 결정되는 div(main-banner-img-wrap)는 img 크기에서 30rem이 줄어든 높이 값을 가지게 됩니다.

```css
.main-banner-img-wrap img {
  width: 100%;
  margin-bottom: -30rem;
}
```

<img width="400" alt="after-margin" src="https://github.com/mixnuts211/Practice-basic-everyday/assets/94048689/2003644e-aeb5-40a8-b4b1-c4d869bc897f">

3.  하지만 이미지는 강제로 잘라내지 않는 이상 부모 밖으로 넘치더라도 자신의 컨텐츠 전체가 화면에 출력됩니다. 이러한 특징을 이용하여 부모의 레이아웃(la-wrap)안에서 변형하고 싶은 레이아웃의 일부분만(main-banner-img-wrap) 재설정을 해줄 수 있습니다.

#### ✔️ **논리 순서대로 마크업하고 CSS로 시각적 스타일링하기**

키보드에 의한 초점은 논리적으로 이동해야 하며 시각적으로 구별할 수 있어야합니다.(웹접근성 검사항목 2.운용의 용이성 중 입력장치 접근성 - 초점 이동) 즉 키보드로 웹사이트를 접근하는 사용자에게 초점이 이동하는 순서는 사용자가 이동하기를 기대하는 순서와 동일하도록 논리적 순서를 고려해야 합니다. 이는 꼭 키보드로 접근하는 사용자뿐 아니라 스크린 리더 사용자들을 위한 것이기도 합니다.
따라서, 마크업 작성 시 기능적으로 논리적인 순서에 맞게 마크업을 작성해야하며, 디자인에서의 시각적 순서가 논리적 순서와 다를 경우, 마크업은 논리적 순서를 따르되 CSS를 활용하여 화면에 보여지는 레이아웃을 재 설정해주어야합니다.

#### ✔️ **button에 이름 부여해주기**

모든 버튼에는 이름 또는 의미가 있어야합니다. 이는 스크린 리더로 웹에 접근할 경우 시각적으로 화면을 보지 않더라도 해당 버튼이 어떠한 역할과 기능을 하는지 전달하기 위함입니다.

방법 1. `sr-only` 활용하기<br>

```html
<button type="button" class="arrow-down">
  <span class="sr-only">메인으로 내려가기</span>
</button>
<!-- 스크린 리더가 해당 button에 접근하게되면 내부의 span 텍스트를 읽어 정보를 전달 -->
```

방법 2. `aria-label`<br>

```html
<button
  type="button"
  class="arrow-down"
  aria-label="메인으로 내려가기"
></button>
<!-- 스크린 리더가 해당 button에 접근하게되면 내부의 aria-label을 읽어 정보를 전달 -->
```

단, 이 경우 시멘틱 태그이거나, role을 사용해 그 역할이 부여되어 상호작용이 가능한 값일 경우에만 사용 가능합니다.
